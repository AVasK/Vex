SIMD stands for Single Instruction Multiple Data and lets your program make use of the vector instructions of your processor.

#include
<immintrin.h>
works for most intrinsics, but not all

#include
<x86intrin.h>
includes ALL intrinsics, works in Clang & GCC compilers

#include
<intrin.h>
includes ALL intrinsics, works in MSVC & ICC compilers


SIMD instruction sets:

  • MMX
  • SSE
  • SSE2
  • SSE3
  • SSE4.1
  • SSE4.2
  • AVX
  • AVX2
  • AVX512

MMX:

A set of 64bit-wide registers fused with x87 FPU
i8
i16
i32
i64

SSE:

A set of 128bit-wide registers. The SSE standard adds 8 x 128-bit registers XMM0 - XMM7 { 16 XMM's (on x86-64) }. XMM's are not aliased with x87 FPU unit, which means that switching between the SSE & the x87 FPU instructions does not imply saving/restoring state information.

Alignment:

16-byte aligned

Types for SSE:

__m128i
-- a 128-bit vector of integer
__m128
-- a 128-bit vector of float
__m128d
-- a 128-bit vector of double
i8
i16
i32
i64

AVX:

A set of 256bit-wide registers, YMM.

Alignment:

32-byte aligned

Types for AVX:

__m256i
-- a 256-bit vector of integer
__m256
-- a 256-bit vector of float
__m256d
-- a 256-bit vector of double
i8
i16
i32
i64
